# MIST · 薄雾

[English](../../README.md) · [中文](README.zh.md) · [日本語](README.ja.md) · [Español](README.es.md) · Русский

MIST — это компактный транспорт частного наложения на основе TLS для надёжного безопасного соединения в корпоративных, коммерческих и самостоятельно управляемых средах, соответствующих нормативным требованиям.

Проект обеспечивает небольшую эксплуатационную поверхность, предсказуемую производительность и понятную конфигурацию для команд, которым требуется внутренний доступ, туннелирование инфраструктуры, удалённое администрирование и связность сервисов в доверенных средах.

Текущая реализация предоставляет:

- Клиент-серверный транспорт на основе TLS
- Мультиплексирование потоков через одно аутентифицированное соединение
- Доступ к локальным приложениям через клиентский SOCKS5/HTTP-прослушиватель
- Кроссплатформенная клиентская библиотека (`mistclient`) для Android, iOS, OpenWRT и встраиваемых систем
- Настраиваемая поддержка填充 пакетов
- Режимы самоподписанных, ACME и пользовательских сертификатов
- Опциональная установка systemd через встроенный установщик
- Целостность кадров HMAC и аутентификация с запросом-ответом (протокол v3)

## Установка Сервера Одной Строкой

```bash
curl -fsSL https://mist.viloris.org/install-server.sh | bash
```

Скрипт определяет вашу архитектуру Linux, загружает последний бинарный файл и устанавливает его в `/usr/local/bin`.

Для интерактивной настройки с сервисом systemd:

```bash
curl -fsSL https://mist.viloris.org/install.sh | bash
```

Предварительно собранные бинарные файлы доступны на странице [Releases](https://github.com/viloris-org/MIST/releases).

## Сборка

```bash
go build ./cmd/mist-server
go build ./cmd/mist-client
```

## Библиотека

Пакет `mistclient/` предоставляет кроссплатформенную клиентскую библиотеку:

```go
import "mist/mistclient"

opts := mistclient.Options{
    ServerAddr: "example.com:8443",
    Password:   "your-password",
    Logger:     myLogger, // реализует mistclient.Logger
}
client, _ := mistclient.NewClient(opts)
defer client.Close()

conn, _ := client.DialStream(ctx, destination)
```

Интерфейс `Logger` позволяет каждой платформе использовать собственное логирование — `android.util.Log` на Android, `os_log` на iOS, `logrus` в CLI. Полная конфигурация описана в `mistclient/options.go`.

## Ручной Запуск Сервера

```bash
./mist-server -l 0.0.0.0:8443 -p "your-password"
```

`-l` задаёт адрес прослушивания. `-p` задаёт общий пароль.

### Режимы Сертификатов

Самоподписанный:

```bash
./mist-server \
  -l 0.0.0.0:8443 \
  -p "your-password" \
  -cert-type self-signed \
  -cert-name 203.0.113.10
```

Если `-cert-name` не указан, сервер определяет его из адреса прослушивания. `0.0.0.0` заменяется на `127.0.0.1`.

ACME:

```bash
./mist-server \
  -l 0.0.0.0:8443 \
  -p "your-password" \
  -cert-type acme \
  -cert-name example.com \
  -acme-http :80 \
  -acme-cache ./cert-cache
```

Пользовательский:

```bash
./mist-server \
  -l 0.0.0.0:8443 \
  -p "your-password" \
  -cert-type custom \
  -cert-file /path/to/cert.pem \
  -key-file /path/to/key.pem
```

## Ручной Запуск Клиента

```bash
./mist-client -l 127.0.0.1:1080 -s example.com:8443 -p "your-password"
```

С закреплением отпечатка самоподписанного сертификата:

```bash
./mist-client \
  -l 127.0.0.1:1080 \
  -s 203.0.113.10:8443 \
  -p "your-password" \
  -tls-cert-sha256 "server-certificate-sha256"
```

С явным указанием SNI при подключении по IP:

```bash
./mist-client \
  -l 127.0.0.1:1080 \
  -s 203.0.113.10:8443 \
  -sni example.com \
  -p "your-password"
```

Клиент также принимает URL `mist://`:

```bash
./mist-client -l 127.0.0.1:1080 -s "mist://password@example.com:8443?sni=example.com"
```

## Примечания по Эксплуатации

- Установите `LOG_LEVEL=debug` для подробного логирования.
- Установите `TLS_KEY_LOG=/path/to/keylog.txt` только на клиенте для отладки TLS.
- Не храните пароли, приватные ключи, скрипты запуска и кэш сертификатов в системе контроля версий.
- Используйте внешние средства управления секретами и жизненным циклом сертификатов для продакшена.

Авторы и контрибьюторы предоставляют это программное обеспечение «как есть» исключительно для авторизованного использования.
