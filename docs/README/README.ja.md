# MIST · ミスト

[English](../../README.md) · [中文](README.zh.md) · 日本語 · [Español](README.es.md) · [Русский](README.ru.md)

MIST（ミスト）は、コンプライアンス準拠のエンタープライズ、商用、自己管理型デプロイメントにおいて、信頼性の高いセキュアな接続を提供するコンパクトなTLSベースのプライベートオーバーレイトランスポートです。

内部アクセス、インフラストラクチャトンネリング、リモート運用、信頼できる環境間のサービス接続を必要とするチーム向けに、小さな運用フットプリント、予測可能なパフォーマンス、明確な設定を提供します。

現在の実装で提供される機能：

- TLSベースのクライアント/サーバー転送
- 単一の認証済みセッション上でのストリーム多重化
- クライアント側SOCKS5/HTTPリスナーを介したローカルアプリケーションアクセス
- Android、iOS、OpenWRT、組み込み向けのクロスプラットフォームクライアントライブラリ（`mistclient`）
- 設定可能なパケットパディング対応
- 自己署名、ACME、カスタム証明書モード
- 付属インストーラーによるオプションのsystemdインストール
- HMACフレーム整合性とチャレンジレスポンス認証（プロトコルv3）

## ワンラインサーバーインストール

```bash
curl -fsSL https://mist.viloris.org/install-server.sh | bash
```

スクリプトがLinuxアーキテクチャを自動検出し、最新のバイナリをダウンロードして `/usr/local/bin` にインストールします。

systemdサービスを含むインタラクティブセットアップ：

```bash
curl -fsSL https://mist.viloris.org/install.sh | bash
```

ビルド済みバイナリは [Releases](https://github.com/viloris-org/MIST/releases) ページから入手できます。

## ビルド

```bash
go build ./cmd/mist-server
go build ./cmd/mist-client
```

## ライブラリ

`mistclient/` パッケージはクロスプラットフォームのクライアントライブラリを提供します：

```go
import "mist/mistclient"

opts := mistclient.Options{
    ServerAddr: "example.com:8443",
    Password:   "your-password",
    Logger:     myLogger, // mistclient.Loggerを実装
}
client, _ := mistclient.NewClient(opts)
defer client.Close()

conn, _ := client.DialStream(ctx, destination)
```

`Logger` インターフェースにより、各プラットフォームが独自のロギングを注入できます — Androidでは `android.util.Log`、iOSでは `os_log`、CLIでは `logrus`。完全な設定は `../../mistclient/options.go` を参照してください。

## サーバーの手動使用方法

```bash
./mist-server -l 0.0.0.0:8443 -p "your-password"
```

`-l` はリッスンアドレス、`-p` は共有パスワードを設定します。

### 証明書モード

自己署名：

```bash
./mist-server \
  -l 0.0.0.0:8443 \
  -p "your-password" \
  -cert-type self-signed \
  -cert-name 203.0.113.10
```

`-cert-name` を省略した場合、サーバーはリッスンアドレスから自動的に導出します。`0.0.0.0` は `127.0.0.1` にフォールバックします。

ACME：

```bash
./mist-server \
  -l 0.0.0.0:8443 \
  -p "your-password" \
  -cert-type acme \
  -cert-name example.com \
  -acme-http :80 \
  -acme-cache ./cert-cache
```

カスタム証明書：

```bash
./mist-server \
  -l 0.0.0.0:8443 \
  -p "your-password" \
  -cert-type custom \
  -cert-file /path/to/cert.pem \
  -key-file /path/to/key.pem
```

## クライアントの手動使用方法

```bash
./mist-client -l 127.0.0.1:1080 -s example.com:8443 -p "your-password"
```

自己署名証明書のフィンガープリントピンニング：

```bash
./mist-client \
  -l 127.0.0.1:1080 \
  -s 203.0.113.10:8443 \
  -p "your-password" \
  -tls-cert-sha256 "server-certificate-sha256"
```

IPで接続する際の明示的なSNI指定：

```bash
./mist-client \
  -l 127.0.0.1:1080 \
  -s 203.0.113.10:8443 \
  -sni example.com \
  -p "your-password"
```

クライアントは `mist://` URLも受け付けます：

```bash
./mist-client -l 127.0.0.1:1080 -s "mist://password@example.com:8443?sni=example.com"
```

## ランタイムノート

- 詳細なログには `LOG_LEVEL=debug` を設定してください。
- TLSデバッグにはクライアント側でのみ `TLS_KEY_LOG=/path/to/keylog.txt` を設定してください。
- パスワード、秘密鍵、起動スクリプト、証明書キャッシュはバージョン管理に含めないでください。
- 本番環境では外部のシークレット管理と証明書ライフサイクルツールを使用してください。
